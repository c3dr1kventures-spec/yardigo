Politique de confidentialité

Version 2.3 · Dernière mise à jour : 20 avril 2026

Cette traduction est fournie à titre informatif. En cas de divergence avec la version néerlandaise, la version néerlandaise fait foi pour les utilisateurs établis aux Pays-Bas. Pour les utilisateurs résidant en Belgique, vos droits sous le RGPD restent intégralement applicables.

YardiGo est une application et un site web néerlandais qui vous permet de trouver et de publier des vide-greniers, des ventes de quartier, des ventes de coffre et des brocantes près de chez vous. Cette politique de confidentialité explique quelles données personnelles nous traitons, pourquoi, pendant combien de temps et sur quelle base juridique, ainsi que les droits dont vous disposez en vertu du Règlement général sur la protection des données (RGPD).

2. Quelles données traitons-nous ?

Nous traitons uniquement les données nécessaires à la prestation du service (minimisation des données, art. 5.1.c RGPD) :

  • Nom – lors de la création d'un compte, comme nom d'affichage dans l'application.
  • Adresse e-mail – pour l'inscription, la connexion et la communication concernant votre compte.
  • Mot de passe haché – via Supabase Auth ; les mots de passe ne sont jamais stockés en clair.
  • Identifiant utilisateur – un code d'identification interne lié à votre compte.
  • Localisation précise (GPS) – pour afficher les ventes près de chez vous et placer votre propre annonce sur la carte. Uniquement après autorisation explicite via votre appareil. Cette localisation n'est pas stockée de manière persistante dans notre base de données ; nous l'utilisons en temps réel pour afficher la carte.
  • Contenu de l'annonce – photos, adresse, date/heure, catégories et description que vous saisissez vous-même lors de la publication d'une vente.
  • Logs techniques – adresse IP, type de navigateur, système d'exploitation et horodatages ; enregistrés automatiquement par Supabase et Vercel pour la sécurité et le diagnostic des erreurs.

4. Conséquences si vous ne fournissez pas les données

  • Sans adresse e-mail et mot de passe, vous ne pouvez pas créer de compte ; vous pouvez toutefois utiliser YardiGo en tant qu'invité pour consulter les ventes.
  • Sans autorisation de localisation, vous ne pouvez pas voir automatiquement les ventes près de chez vous ; la recherche manuelle par ville ou code postal reste possible.
  • Sans données d'annonce (adresse, date), vous ne pouvez pas publier de vente ; consulter les ventes des autres reste possible.

5a. Ventes de quartier : adresses d'autres résidents

Lors d'une vente de quartier, l'organisateur peut indiquer plusieurs adresses dans le formulaire de publication. L'adresse d'un autre ménage est également une donnée personnelle de ce(s) résident(s). Pour ce traitement, une base juridique distincte est nécessaire. YardiGo travaille donc avec un lien de confirmation obligatoire :

  • L'adresse personnelle de l'organisateur est mise en ligne directement. Pour cela, l'organisateur donne explicitement son consentement pour lui-même au moment de la publication (art. 6.1.a RGPD).
  • Pour chaque autre adresse, YardiGo génère un lien unique et aléatoire (/bevestig?token=…) et ne l'envoie pas directement au résident. L'organisateur partage le lien manuellement avec le résident concerné via WhatsApp, SMS ou e-mail.
  • Tant que le résident ne clique pas sur le lien, l'adresse n'est pas visible aux autres utilisateurs (même pas aux visiteurs connectés). L'annonce reste au statut en attente.
  • Le résident peut cliquer via le lien sur "Oui, confirmer mon adresse" (consentement ; statut confirmé) ou "Non, retirer" (refus ; statut refusé). En cas de refus, la rue, le code postal et les coordonnées sont immédiatement supprimés de notre base de données.
  • Si un lien de confirmation reste sans réponse pendant 7 jours, YardiGo bascule automatiquement le statut à expiré et la rue, le code postal et les coordonnées sont également supprimés. Le résident n'a rien à faire.
  • Le résident peut éventuellement saisir un nom d'affichage (par ex. "Famille de Vries") ; cela n'est jamais obligatoire.
  • Si en tant que résident vous estimez que votre adresse a été saisie sans votre accord, envoyez un e-mail à privacy@yardigo.nl. Nous la supprimons sur simple demande, quel que soit le statut.

Dans ce processus, nous traitons les données suivantes du résident : nom de rue + numéro, code postal, coordonnées géographiques et (facultatif) le nom d'affichage que le résident a choisi. Nous ne traitons aucun numéro de téléphone ni adresse e-mail du résident. La communication avec le résident se fait entièrement en dehors de YardiGo, par l'intermédiaire de l'organisateur.

Étiquette de reconnaissance par l'organisateur. Lors de la création d'une vente de quartier, l'organisateur peut éventuellement saisir une courte étiquette de reconnaissance par adresse (par ex. un prénom ou "n° 14"). Cette étiquette sert uniquement d'aide-mémoire pour l'organisateur lui-même, n'est visible que dans son tableau de bord et n'apparaît jamais publiquement sur la carte, la page de détails ou la page de confirmation. Si un résident saisit son propre nom d'affichage via le lien de confirmation, cela ne remplace pas l'étiquette de l'organisateur, mais apparaît séparément dans le profil de résident de l'annonce.

L'adresse e-mail du voisin est facultative et n'est pas stockée. L'organisateur peut saisir une adresse e-mail par adresse dans le formulaire de publication, uniquement pour ouvrir le bouton e-mail sur la page de partage pré-rempli directement après publication (via un lien mailto:). L'adresse e-mail est utilisée uniquement dans le navigateur de l'organisateur, ne passe pas par notre serveur et n'est pas stockée dans notre base de données. L'e-mail lui-même est envoyé par l'application e-mail de l'organisateur depuis sa propre adresse e-mail ; YardiGo n'est pas expéditeur ni sous-traitant dans cet envoi.

Base juridique : consentement explicite via le lien de confirmation (art. 6.1.a RGPD). Le consentement peut être retiré à tout moment via ce même lien (tant que la vente n'a pas eu lieu) ou via privacy@yardigo.nl. Le retrait n'a pas d'effet rétroactif sur le traitement effectué avant le retrait.

Durée de conservation : active jusqu'à la date de la vente de quartier comprise ; ensuite les données d'adresse, de code postal et de coordonnées des résidents confirmés (plus l'étiquette de reconnaissance optionnelle de l'organisateur) sont supprimées dans les 30 jours. Les entrées en attente ou refusées sont supprimées immédiatement ou après 7 jours, comme décrit ci-dessus.

6. Cookies et stockage local

YardiGo n'utilise aucun cookie de traçage, aucun service d'analyse contenant des données personnelles (comme Google Analytics ou Meta Pixel) ni aucun pixel publicitaire. Nous ne plaçons que :

  • localStorage technique – pour stocker votre statut de connexion et vos préférences (comme les catégories sélectionnées) sur votre appareil. Il ne s'agit pas de cookies et les données ne quittent pas votre appareil.
  • Cookies de session fonctionnels de Supabase Auth – strictement nécessaires pour vous garder connecté. Pour les cookies strictement nécessaires, le consentement n'est pas requis (art. 82 de la loi française "Informatique et Libertés" et législation belge équivalente transposant la directive ePrivacy).

Comme nous ne chargeons aucun script analytique ou marketing, nous n'affichons délibérément aucune bannière de cookies. Si à l'avenir YardiGo utilise des analytiques ou d'autres cookies non nécessaires, nous placerons à ce moment-là une bannière de consentement avec des options "Accepter" et "Refuser" équivalentes, et ces scripts ne seront chargés qu'après consentement.

7. Sécurité

  • Toute communication avec notre serveur se fait via HTTPS (TLS).
  • Les mots de passe ne sont jamais stockés en clair ; nous utilisons l'implémentation de hachage de Supabase Auth.
  • L'accès à la base de données est limité par Row Level Security (RLS) sur Supabase : vous ne pouvez modifier que vos propres annonces et données de profil.
  • Une limitation de débit (rate-limiting) est active sur les endpoints de connexion et d'inscription pour limiter les tentatives de brute-force.

8. Décisions automatisées et profilage

YardiGo n'effectue aucune prise de décision automatisée ni profilage au sens de l'article 22 du RGPD. Aucune décision vous concernant n'est donc prise par des algorithmes ayant des effets juridiques ou des conséquences significatives pour vous.

9. Vos droits

Sous le RGPD, vous disposez des droits suivants :

  • Accès (art. 15) – demander quelles données nous détenons à votre sujet.
  • Rectification (art. 16) – faire corriger des données inexactes.
  • Suppression / oubli (art. 17) – faire supprimer votre compte et les données associées. Vous pouvez le faire vous-même via Mes ventes → Paramètres → Supprimer le compte dans l'application, ou par demande à privacy@yardigo.nl.
  • Limitation (art. 18) – nous demander de limiter temporairement le traitement, par exemple en cas de litige sur l'exactitude.
  • Portabilité (art. 20) – récupérer vos données de compte et annonces dans un format structuré et lisible par machine.
  • Opposition (art. 21) – vous opposer aux traitements basés sur l'intérêt légitime (notamment les logs).
  • Retrait du consentement (art. 7.3) – l'autorisation GPS peut être retirée à tout moment via les paramètres de votre appareil. Cela n'affecte pas le traitement déjà effectué avant le retrait.

Envoyez votre demande à privacy@yardigo.nl. Nous répondons dans un délai d'un mois (art. 12.3 RGPD). Pour vérifier votre identité, nous vous demandons d'envoyer la demande depuis l'adresse e-mail enregistrée pour votre compte ; une copie d'une pièce d'identité n'est pas nécessaire.

11. Utilisateurs de moins de 16 ans

YardiGo est destiné aux utilisateurs à partir de 16 ans. Les utilisateurs de moins de 16 ans ont besoin du consentement d'un parent ou tuteur en vertu de l'article 8 du RGPD. Si vous soupçonnez que nous traitons les données d'un mineur sans autorisation, contactez-nous via privacy@yardigo.nl ; nous supprimerons ces données dans les meilleurs délais.

12. Violations de données

Nous prenons au sérieux notre obligation de notification en cas de violation de données. Si une violation de données présente un risque pour vos droits et libertés, nous la notifions à l'Autoriteit Persoonsgegevens dans les 72 heures après sa découverte (art. 33 RGPD). En cas de risque élevé, nous vous informons directement (art. 34 RGPD).
